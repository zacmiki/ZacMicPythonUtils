def normalize_Opus_xArrays(sample, reference):
    import xarray as xr
    
    sample_x = sample.x.values
    
    # handle if reference is a numpy array
    if isinstance(reference, xr.Dataset) or isinstance(reference, xr.DataArray):
        reference_x = reference.x.values
        ref_values = reference.ssc.values
    else:
        reference_x = sample_x  # assume same x-axis
        ref_values = reference  # use directly

    )
    return normalized_dataset

Excellent choice interactive 3D helps you see clustering or group separation that might not appear in 2D PCA.

Your axes (PLS1, PLS2, PLS3) correspond to the first three latent variables (LV1LV3) of PLS-DA.

Why 3D can be better here

Unlike PCA (which maximizes overall variance), PLS-DA maximizes class separation, so even the third component (PLS3) can meaningfully contribute to discrimination between groups.

A 3D plot helps:

  • Visualize overlaps between classes,

  • Detect possible outliers,

  • Interpret subtle separations that 2D projections might hide.

Optional enhancements

Show explained variance per component

You can calculate how much of the X variance each PLS component explains:

python
explained = 100 * np.var(X_scores, axis=0) / np.var(X_scaled, axis=0).sum()
print(f"Explained variance: PLS1={explained[0]:.2f}%, PLS2={explained[1]:.2f}%, PLS3={explained[2]:.2f}%")

(Note: unlike PCA, this is approximate since PLS optimizes covariance with Y, not total variance.)

Add loadings (spectral contributions)

You can extract and inspect loadings (like PCA loadings) to see which wavenumbers drive class separation:

python
wavenumbers = rbw1_list[0]["x"].values
pls_loadings = pls.x_loadings_

plt.figure(figsize=(10, 6))
for i in range(3):
    plt.plot(wavenumbers, pls_loadings[:, i], label=f"PLS{i+1}")
plt.xlabel("Wavenumber (cm)")
plt.ylabel("PLS Loading Weight")
plt.title("PLS-DA Loadings (Spectral Contribution)")
plt.legend()
plt.grid(True)
plt.gca().invert_xaxis()
plt.show()

Bonus: Class separation check

To confirm the model really separates classes (not overfitting), use a simple cross-validation:

python
from sklearn.model_selection import cross_val_score

cv_scores = cross_val_score(pls, X_scaled, Y, cv=5, scoring="r2")
print("Cross-validation R scores:", cv_scores)
print("Mean R:", np.mean(cv_scores))
